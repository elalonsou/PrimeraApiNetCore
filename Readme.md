# Primera API Net Core
El objetivo de este proyecto es poner en práctica los diferentes patrones aprendidos y 
que considere interesantes.
No hay una hoja de ruta establecida y el proyecto puede sufrir fuertes cambios de rumbo si se considera
oportuno.

Como objetivo se plantea crear un web API para el Back-End de una aplicación que permita
planificar recetas.

A continuación, se describen las arquitecturas y patrones que se utilicen:
- Entity Framework Core
- Patrón repositorio
- Patrón capas (capa datos y capa negocio (web API))
- Funcionamiento asíncrono
- Loggin de ASP Net Core
- AutoMapper

Se quiere incluir:
- Swagger
- Pruebas unitarias
- Sistema de Autenticacion/Autorización (¿OAuth?)
